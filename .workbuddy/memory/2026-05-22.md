### 技术细节
- GitHub Actions cron 语法：`0 2,6 * * 1-5` = 每周一至周五的 UTC 02:00 和 06:00
- 时区对照：UTC+8 = 北京时间，所以 UTC 02:00 = 北京 10:00，UTC 06:00 = 北京 14:00
- 选择这两个时间点的原因：10:00 是开盘后1小时数据已稳定，14:00 是尾盘前1小时适合尾盘决策
